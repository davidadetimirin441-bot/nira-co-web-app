NIRA & CO UK source - deploy guide

Recommended simple launch path:
1. Put this whole nira-co-web-app folder into a GitHub repository.
2. Create a Web Service on Render, Railway, Fly.io, or another Python host.
3. Use:
   Build command: pip install -r requirements.txt
   Start command: HOST=0.0.0.0 python server.py
4. Add your custom domain: niraandco.co.uk.
5. Make sure HTTPS is active.

Mobile access:
- iPhone: open the HTTPS website in Safari, tap Share, then Add to Home Screen.
- Android: open the HTTPS website in Chrome, then install/add to home screen when prompted.

App stores later:
- Google Play: package the website as a Trusted Web Activity or convert to a native shell.
- Apple App Store: use a native wrapper or rebuild the client in React Native/Swift.

Production upgrades before real users:
- Replace SQLite with hosted Postgres.
- Keep SECRET_KEY private so password reset and verification token hashes cannot be recreated by end users.
- Add Google/Apple/Microsoft OAuth.
- Add a licensed property-feed API.
- Connect Stripe or another payment processor for real £15/month billing.
- Connect Airtim and UK bank/Open Banking providers before accepting real payments.
- Add your own business bank account through the payment provider dashboard so subscription money pays out to you.
- Manual bank-transfer details have been removed. Use Stripe Checkout and Stripe payouts for production subscriptions.
- For money from traffic, add a real ad network, affiliate programme, sponsored listing model, or lead fee model. Visits by themselves do not create income.
- For Google Ads/AdSense, get niraandco.co.uk approved by Google, then add the publisher ID and ad slot in the Ad earnings page.
- Connect a production AI provider for live assistant responses.
- Connect SMTP, SendGrid, Mailgun, Postmark, or another email service for real reset-email delivery.
- Use the in-app newsletter tool weekly, or automate POST /api/newsletter/send from your hosting provider once authentication/admin controls are added.
- Set ADMIN_EMAILS to a comma-separated list of trusted admin account emails. Do not grant admin access through the browser.
- Password reset links open the dedicated /reset?token=... page.
- Set BASE_URL to your live website, https://nireco.co.uk, so password reset and verification links generate automatically with the correct domain.

SMTP variables for password reset emails:
- BASE_URL
- SECRET_KEY
- COOKIE_SECURE
- ADMIN_EMAILS
- SMTP_HOST
- SMTP_PORT
- FROM_EMAIL
- SMTP_USERNAME
- SMTP_PASSWORD
- SMTP_TLS
- Password reset and email verification are part of Stage 1.
- Add backups, monitoring, and terms/privacy pages.
